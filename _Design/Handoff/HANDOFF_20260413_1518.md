**수정 파일**: EquipmentSubsystem.h/.cpp, RSPlayerController.h/.cpp (파일명 주의: RS prefix)

**주요 작업**:
1. `SLOT_COUNT = 2` 상수 변경 (EquipmentSubsystem)
2. `RequestManualFire()` 제거 (호출처까지 전부)
3. `FindNearestEnemy()` 추가 — 일정 범위 내 최근접 적 FHitResult 반환
4. `StartAutoFire()` / `FireSlot(int32 SlotIndex)` — 찾은 적을 타겟으로 ProjectileAttack GA 발동
5. RSPlayerController: `IA_Attack / Slot1 / Slot2 / Slot3` 바인딩 제거
6. RSPlayerController: `IA_SkillQ / IA_SkillE` UPROPERTY 추가 (바인딩은 M-5에서)

**엣지 케이스**:
- 범위 내 적 없을 때 → FireSlot 스킵 (조용히)
- 적이 죽었을 때 타겟 재탐색 트리거 필요
- SLOT_COUNT 변경 시 기존 SlotIndex 2 참조 코드 전부 확인

### [MODULE-4] ExecCalc 데미지 공식 [P0]
> 실제 ATK/DEF 수식으로 GE 교체.

**신규 파일**: RS_DamageExecCalc.h/.cpp
**수정 파일**: RSGameplayTags.h, GA_ProjectileAttack.cpp, (에디터) GE_WeaponDamage.uasset, GE_EnemyDamage.uasset

**주요 작업**:
1. `Data.WeaponBaseDamage` / `Data.EnemyAttackDamage` SetByCaller 태그 추가 (RSGameplayTags.h)
2. `URS_DamageExecCalc : UGameplayEffectExecutionCalculation` 신규
   - 플레이어→에너미: `(ATK × DamageMultiplier) × (1 - DEF / (DEF + 100))`
   - 에너미→플레이어: 동일 공식 (역방향)
3. GA_ProjectileAttack: `SetByCallerMagnitude(Data.WeaponBaseDamage, WeaponATK)` 주입
4. **에디터 작업**: GE_WeaponDamage / GE_EnemyDamage Executions 탭에 RS_DamageExecCalc 연결

**주요 작업**:

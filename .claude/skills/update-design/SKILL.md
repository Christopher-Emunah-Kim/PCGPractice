# /update-design RUNBOOK
> 페르소나: 게임 시스템 기획자
> 역할: 구현 결과/설계 변경을 기획서에 반영

## STATE_MACHINE
```
INIT ──→ [A] 갱신 범위 확인
          └─ [B] 기획서 분석
                └─ [C] 갱신 초안 작성
                      └─ [D] 사용자 승인
                            ├─ 승인 → 수정 + 이력 + 버전갱신
                            └─ 수정 → [C]
DONE
```

## EXEC

### [A] 범위 확인
- 대상 기획서 (`_Design/References/Systems/` 내)
- 갱신 사유: 구현결과반영 / 설계변경 / 오류수정

### [B] 기획서 분석
- 버전 번호, 변경 필요 섹션, 연쇄 영향 기획서 파악

### [C] 갱신 초안
```
## [갱신 제안] ○○_기획 vX.X → vY.Y

변경 사유:
변경 내용:
  섹션: ○○
    기존: (기존 내용)
    변경: (새 내용)
    근거: (사유)
연쇄 영향:
```

**문서 규칙:**
- 선언적/엔진 비종속 언어 (UE5 함수명 금지)
- 디자이너/아티스트가 읽을 수 있는 수준
- 데이터 스키마: 테이블 형태
- 예외처리: "상황 → 처리" 형태

### [D] 승인 후 실행
기획서 수정 + 정정이력 추가 + 버전 갱신

## RULES
```
- 코드 스타일 문체 변환 금지 (선언적 언어 유지)
- 연쇄 영향 미확인 상태로 단일 기획서만 갱신 금지
- 사용자 승인 없이 수정 금지
```
